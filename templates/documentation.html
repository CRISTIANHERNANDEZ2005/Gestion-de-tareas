Documentación Técnica del Proyecto

¡Hola! Si estás leyendo esta documentación, probablemente eres un desarrollador que quiere entender cómo construí este Gestor de Tareas. Te explicaré cómo desarrollé cada parte del proyecto, desde la arquitectura hasta los detalles de implementación, para que puedas mantenerlo o extenderlo fácilmente.

Frontend Modular

Implementé una arquitectura basada en módulos ES6 para organizar mejor el código JavaScript y facilitar su mantenimiento.

Backend Robusto

Desarrollé una API RESTful con PHP y MySQL para gestionar eficientemente los datos de las tareas.

Diseño Responsivo

Creé una interfaz completamente adaptable a diferentes dispositivos para que los usuarios puedan acceder desde cualquier lugar.

Mi Enfoque de Desarrollo

Cuando desarrollé este proyecto, me aseguré de seguir buenas prácticas de programación, manteniendo un código limpio, bien documentado y fácil de entender. Mi objetivo fue crear una aplicación funcional pero también escalable para futuras mejoras.

Arquitectura del Sistema

Para este proyecto, elegí una arquitectura cliente-servidor tradicional con una clara separación entre frontend y backend. Esta decisión me permitió mantener el código organizado y facilitar el mantenimiento futuro.

Frontend (Cliente)

En el frontend, decidí utilizar tecnologías web estándar pero con un enfoque moderno:

  • HTML5: Estructuré la interfaz con etiquetas semánticas para mejorar la accesibilidad y SEO.
  • CSS3: Creé estilos personalizados con un enfoque en la experiencia de usuario y el diseño responsivo.
  • JavaScript (ES6+): Implementé interactividad y comunicación con el backend usando las últimas características de JavaScript.
  • Módulos ES6: Organicé el código en módulos reutilizables para mantener una estructura limpia y evitar el código espagueti.

Backend (Servidor)

Para el backend, opté por tecnologías robustas y bien establecidas:

  • PHP 7+: Desarrollé la lógica del servidor y la API RESTful aprovechando las mejoras de rendimiento de PHP 7.
  • MySQL: Elegí MySQL como sistema de gestión de bases de datos por su confiabilidad y facilidad de uso.
  • PDO: Utilicé PDO para acceder a la base de datos de forma segura y prevenir inyecciones SQL.
  • JSON: Implementé JSON como formato de intercambio de datos entre frontend y backend para una comunicación eficiente.

Base de Datos que Diseñé

Para la base de datos, creé una estructura simple pero eficiente que cumple con todos los requisitos del proyecto:

Script de Creación

Este es el script que desarrollé para crear la base de datos:

-- Creación de la base de datos
CREATE DATABASE gestor_tareas_db;

-- Uso de la base de datos
USE gestor_tareas_db;

-- Creación de la tabla tareas
CREATE TABLE tareas (
    id INT AUTO_INCREMENT PRIMARY KEY,
    titulo VARCHAR(100) UNIQUE NOT NULL,
    descripcion TEXT,
    fecha_limite DATE,
    creado_en TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
Campo Tipo Nulo Clave Predeterminado Extra
id INT NO PRI NULL auto_increment
titulo VARCHAR(100) NO UNI NULL
descripcion TEXT YES NULL
fecha_limite DATE YES NULL
creado_en TIMESTAMP NO CURRENT_TIMESTAMP

Decisiones de Diseño

Cuando diseñé esta estructura, tomé varias decisiones importantes:

  • PRIMARY KEY: Usé un id autoincremental para identificar de forma única cada tarea.
  • UNIQUE: Hice que el título sea único para evitar duplicados y confusiones.
  • TIMESTAMP: Agregué un campo de creación automática para llevar registro de cuándo se creó cada tarea.

API Endpoints que Desarrollé

Creé una API RESTful con endpoints específicos para cada operación CRUD:

GET /php/obtener_tareas.php

¿Qué hace? Este endpoint que desarrollé obtiene todas las tareas almacenadas en la base de datos, ordenadas por fecha límite.

POST /php/agregar_tarea.php

¿Qué hace? Este endpoint que creé permite crear una nueva tarea en la base de datos.

Parámetros

  • titulo (requerido): Título de la tarea (5-100 caracteres)
  • descripcion (opcional): Descripción detallada
  • fecha_limite (requerido): Fecha límite en formato YYYY-MM-DD

Respuesta

{
  "exito": true,
  "mensaje": "Tarea agregada correctamente."
}

POST /php/actualizar_tarea.php

¿Qué hace? Con este endpoint que desarrollé puedes actualizar una tarea existente en la base de datos.

Parámetros

  • id (requerido): ID de la tarea a actualizar
  • titulo (requerido): Nuevo título de la tarea
  • descripcion (opcional): Nueva descripción
  • fecha_limite (requerido): Nueva fecha límite

Respuesta

{
  "exito": true,
  "mensaje": "Tarea actualizada correctamente."
}

POST /php/eliminar_tarea.php

¿Qué hace? Este endpoint que implementé elimina una tarea de la base de datos.

Parámetros

  • id (requerido): ID de la tarea a eliminar

Respuesta

{
  "exito": true,
  "mensaje": "Tarea eliminada correctamente."

Módulos JavaScript que Creé

Para mantener el código frontend organizado, desarrollé una arquitectura modular usando ES6 Modules:

formValidator.js

¿Qué hace? Este módulo que desarrollé se encarga de toda la validación de formularios y manejo de errores.

Funciones que Creé

  • mostrarError(input, mensaje): Muestra mensaje de error debajo de un campo
  • limpiarError(input): Limpia mensaje de error de un campo
  • validarCampoTexto(input, opciones): Valida campos de texto según criterios
  • validarFecha(input, opciones): Valida campos de fecha
  • verificarTituloUnico(titulo, tareaId): Verifica unicidad de títulos vía AJAX

Características que Implementé

  • Animaciones suaves para mostrar/ocultar errores
  • Validación en tiempo real durante la escritura
  • Verificación asíncrona de unicidad de títulos
  • Configuración flexible de reglas de validación

messageHandler.js

¿Qué hace? Este módulo que creé maneja todas las notificaciones y mensajes al usuario.

Funciones que Desarrollé

  • mostrarMensajeFlotante(mensaje, tipo): Muestra notificaciones flotantes
  • crearMensaje(mensaje, tipo): Crea elementos de mensaje en DOM
  • cerrarMensaje(event): Cierra mensajes interactivos

Características que Implementé

  • Mensajes de éxito, error y advertencia
  • Animaciones de entrada/salida suaves
  • Auto-eliminación después de 5 segundos
  • Cierre manual con botón interactivo

taskManager.js

¿Qué hace? Este módulo que desarrollé gestiona todas las operaciones CRUD de tareas.

Funciones que Creé

  • eliminarTarea(): Elimina tarea seleccionada mediante AJAX
  • ocultarModalEliminacion(): Cierra modal de confirmación
  • setIdTareaAEliminar(id): Establece ID de tarea a eliminar
  • getIdTareaAEliminar(): Obtiene ID de tarea a eliminar

Características que Implementé

  • Integración con sistema de mensajes
  • Manejo de estado de tarea a eliminar
  • Uso de eventos personalizados para actualización
  • Comunicación asíncrona con backend

Cómo Configuré el Proyecto

Si quieres ejecutar este proyecto localmente, sigue estos pasos que yo mismo seguí para configurarlo:

1. Requisitos Previos

Antes de comenzar, asegúrate de tener instalado:

  • Servidor web local (Apache, Nginx, etc.) - Yo usé XAMPP
  • PHP 7.0 o superior - Mi versión recomendada es PHP 7.4+
  • MySQL 5.6 o superior - Yo usé MySQL 5.7
  • Navegador web moderno (Chrome, Firefox, Edge) - Yo desarrollo y pruebo en Chrome

2. Instalación del Proyecto

Sigue estos pasos que yo seguí para instalar el proyecto:

  1. 1 Cloné o descargué el repositorio del proyecto en el directorio raíz de mi servidor web (por ejemplo, htdocs en XAMPP).
  2. 2 Accedí a phpMyAdmin o herramienta similar para gestionar la base de datos MySQL.
  3. 3 Ejecuté el script SQL ubicado en sql/database.sql para crear la base de datos y la tabla necesarias.
  4. 4 Verifiqué la configuración de conexión en php/conexion.php y ajusté según mis credenciales locales.
  5. 5 Inicié el servidor web y accedí al proyecto a través de mi navegador (por ejemplo, http://localhost/gestor_tarea).

3. Configuración de la Base de Datos

En mi archivo de conexión, configuré estos valores:

// php/conexion.php
define('DB_HOST', 'localhost');
define('DB_NOMBRE', 'gestor_tareas_db');
define('DB_USUARIO', 'root');
define('DB_CONTRASENA', '');

Asegúrate de actualizar estos valores según tu configuración local de MySQL.

4. Verificación

Una vez completados los pasos anteriores, deberías poder:

  • Acceder a la aplicación en tu navegador
  • Crear nuevas tareas
  • Ver la lista de tareas existentes
  • Editar y eliminar tareas
